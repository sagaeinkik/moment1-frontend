Arbetsprocess

För att lösa uppgiften har jag först tagit kameran (Lumix GH5) och tagit bilder på random saker i lägenheten. Jag har bildbehandlat och sparat ner till 2000x1127(ish) men har undvikit att komprimera filerna. Samtliga filer har sparats i jpg-format och har legat på en storlek på 1-2 MB. I övrigt har projektet gått som följande:

  1. Jag skapade en mapp för hela mitt projekt, med en undermapp vid namn source. I den skapade jag tre HTML-filer med emmet-mallar, samt lade in mappen med mina grafikfiler. Jag skapade också en mapp för CSS och skapade en CSS-fil.
  2. I terminalen körde jag kommandot npm init och sedan npm install parcel --save-dev.
  3. I package.json-filen som skapas av parcel ändrade jag i script såhär: "script" { "start": "parcel", "build": "parcel build" }.
  4. I terminalen skrev jag kommandot npm start för att sätta igång byggandet.
  5. Jag initierade ett nytt git-repo med git init. I terminalen skrev jag också $ echo .DS_STORE > .gitignore för att skapa och lägga till en rad automatiskt. Sedan öppnade jag filen med $ open .gitignore för att också lägga till .parcel-cache, node_modules och dist. Dist lade jag till för att Mattias nämnde det som hastigast i videon under Teori och läsanvisningar för momentet.
  6. Jag skrev kommandot git checkout -b development för att skapa en ny branch och byta till den i samma veva.
  7. Vid det här stadiet hade jag börjat bygga mina HTML-filer och CSS-filen samt en liten JavaScript-funktionalitet för menyknappen. Jag gjorde regelbunda commits lokalt.
  8. När jag kände att jag hade kommit till en bra punkt skapade jag ett nytt repo på Github, och i terminalen skrev jag $ git add remote (url:en). När det var klart skrev jag $ git push origin development.
  9. För att optimera bilderna har jag, som i exemplet i Parcels dokumentation, använt mig av <picture>-taggen. I den finns <source>-taggar med media queries, och sökvägen till bilden har tilläggen "?as=(filändelse)&width=(storlek)". Jag har använt 2 olika storlekar, med tre olika filformat.
  10. För att lösa bakåtkompabiliteten utgick jag återigen från dokumentationen (klicka här) och lagt till "browserslist": "> 0.5%, last 2 versions, not dead" i package.json-filen.

I övrigt ska Parcel, som jag har förstått det, sköta mycket automatiskt. Jag behöver alltså inte lägga till någonting för att filerna ska minifieras vid publicering.

Jag är osäker på bakåtkompabiliteten och att slå ihop flera filer (t ex js eller css). Jag vågade heller inte använda scss som jag brukar använda (vilket jag ångrar nu – vad jobbigt det är att gå tillbaka till vanlig css efter att man har vant sig vid nesting!) Men GUD vilken skillnad att inte behöva komprimera bilderna själv! Nu hoppas jag bara att jag har gjort någorlunda rätt.
