Arbetsprocess

För att lösa uppgiften har jag först tagit kameran (Lumix GH5) och tagit bilder på random saker i lägenheten. Jag har bildbehandlat och sparat ner till 2000x1127(ish) men har undvikit att komprimera filerna. Samtliga filer har sparats i jpg-format och har legat på en storlek på 1-2 MB.

Med hjälp av Mattias video från momentets teorisektion har jag sedan skapat en mapp för hela mitt projekt och en undermapp som heter "Source". I source skapade jag tre html-filer och lade in mina grafikfiler. Sedan körde jag kommandot npm init och sedan npm install parcel --save-dev. Jag ändrade i script till att se ut som script { "start": "parcel", "build": "parcel build" } och sedan var det bara att köra igång med kommandot npm start.

Direkt efteråt initierade jag ett nytt git-repo och har gjort lokala commits till en development-branch. Efter att startsidan och bilder-sidan kommit på plats skapade jag ett motsvarande repo på github och länkade samman dessa med git remote add (url). Dit pushar jag när jag känner att jag har kommit en bit på vägen.

För att optimera bilder har jag, som i exemplet som anges i Parcels dokumentation, använt mig av <picture>-taggen. I den finns <source>-taggar med media queries, och sökvägen till bilden har tilläggen "?as=(filändelse)&width=(storlek)".

Parcel sköter det mesta självmant, som minifieringen av HTML, CSS och JavaScript. För att lösa bakåtkompabilitet har jag även här utgått från dokumentationen (klicka här) och lagt till "browserslist": "> 0.5%, last 2 versions, not dead" i min package.json-fil.

Jag är osäker på bakåtkompabiliteten och att slå ihop flera filer (t ex js eller css). Jag vågade heller inte använda scss som jag brukar använda. Men GUD vilken skillnad att inte behöva komprimera bilderna själv! Nu hoppas jag bara att jag har gjort någorlunda rätt.
